BLOCK

Hello World
  • <div> es un elemento de tipo BLOCK.
  • De forma predeterminada, los elementos de BLOCK comienzan en una nueva línea. (Fuerzan un salto de línea)
  • Toman todo el ancho disponible (Se extienden de izquierda a derecha tanto como sea posible)
  • Respetan el width, height, margin-top y margin-bottom indicados por el usuario.
  • Una forma de ver si un elemento es BLOCK: escribir varios seguidos y comprobar si se fuerza el salto de línea.
    Si los elementos quedan uno al lado del otro entonces pueden ser de tipo INLINE o INLINE-BLOCK.

INLINE

Hello World
  • <span> es un elemento de tipo INLINE.
  • Los elementos INLINE pueden comenzar en cualquier parte de una línea.
  • No fuerzan un salto de línea.
  • No tienen margin-top ni margin-bottom, pero si tienen margin-left y margin-right.
  • No respetan width ni height. Estas medidas dependerán del tamaño en píxels de su contenido.
  • Una forma de ver si un elemento es INLINE:es darle ancho. Si asignas width:100% y la caja no
    ocupa todo el ancho disponible, se trata de un elemento INLINE.

Este es un elemento <p> (BLOCK)

Hello World

  • Los elementos BLOCK pueden contener elementos INLINE y otros elementos BLOCK. La idea de que los elementos en BLOCK
    crean estructuras "más grandes" que los elementos INLINE.
  • Se observa que el estilo deja de aplicarse a la ultima linea del parrafo y se presume que es porque en medio hay otro elemento de tipo BLOCK
  • Como cada cuadro ocupa el total del ancho de la pantalla, se colocan uno debajo del otro, recordemos que eso hacen los elementos de tipo BLOCK. Pero, como el de arriba no contiene nada, se ve plano.

CONCLUSIONES FINALES:

Es sumamente importante saber diferenciar estos 2 tipos de elementos ya que de ello desprende un código limpio y bien estructurado; sin contar que sabiendo esto nos podemos evitar horas de debugging sobre "errores fantasma de alineación".
  • El tipo de elemento definirá como se va a comportar un elemento con respecto a los que lo rodean.
  • Los elementos BLOCK se van colocando verticalmente forzando un salto de linea al final de cada uno.
  • Los elementos BLOCK respetan el width, el height y los margenes verticalesque se les asigna.
  • Las anchura es la máxima que puede tomar dentro de su elemento contenedor.
  • Puede contener otros elementos de tipo inline y block.
  • Los elementos INLINE se van apilando en horizontal sin hacer saltos de linea.
  • Los elementos INLINE hacen caso omiso del width y height que les indicamos. Estos se definen en base al contenido que posea.
  • Los elementos INLINE no se ven afectados por el flujo de otros elementos, limitándose solamente a su uso natural. Por el contrario los elementos BLOCK no se mezclan con el resto de los elementos, mostrándose en líneas independientes.
  • Solo puede contener elementos de tipo inline.
  • Los elementos BLOCK modifican el comportamiento de los otros elementos que le rodean.
